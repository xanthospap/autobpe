NAME

wgeturanus − AutoBpe Software Collection

INTRODUCTION

AutoBpe Software Collection is a collection of tools that accomodates automatic processing of GNSS data mainly via Bernese GNSS.

SYNOPSIS

wgeturanus [OPTIONS]... −y [YEAR] −d [DAY OF YEAR]

DESCRIPTION

Download RINEX files (Hatanaka compressed) for URANUS network for a specific date. The data center used in this script to download the files is Tree-Company. The available stations for downloading can be listed using the −p option. For this script to work, a station information file must be provided, with the full name and 4-char ids of the URANUS stations. By default, the script will search for a file named uranus.tbl found in running directory (see −l option).

Mandatory arguments for long options are mandatory for short options too.
−s, −−stations

specify stations to download seperated by whitespace (e.g -s penc pdel ...), see STATION PRIORITY

−f, −−station-file

give a file specifying stations to be downloaded. The file must have one line, where each station is seperated by a whitespace character, see STATION PRIORITY

−y, −−year

specify year (4-digits)

−d, −−doy

specify day of year

−o, −−output-directory

specify output directory where the downloaded files are going to be saved (default ./)

−m, −−max-stations

specify a station limit. When this limit is reached, no more stations are going to be downloaded.

−r, −−force-remove

if a file is found with the same name as the one to be downloaded, it will be deleted before downloading. The default behaviour is that if a file is found with the same name as the one to be downloaded, then the file will NOT be downloaded but counted as downloaded.

−z, −−decompress

decompress the downloaded files (default do not decompress). If this switch is not enabled and a file is found matching a file to be downloaded but not compressed, then the existing file is going to be compressed (to match exactly the file as it would be downloaded).

−n, −−upper-case

truncate all downloaded files to upper case letters.

−q, −−quiet

supress all messages, except for errors.

−h, −−help

only print the help message and exit

−x, −−fix-header

fix header (MARKER NAME field in RINEX) to match the station’s name (4-char ID)

−p, −−print-stations

list all URANUS gnss stations which can be downloaded via this script and exit

−l, −−table-file

specify the station information table; default uranus.tbl in running directory.The format of the table must be:
Station name URL DSO
ID ID
--- --------------- ---- ----
001 Ag.Theodoroi agth agth
002 AgNikolaos agni agni
090 Zakunthos zaku zaku

where Station name is the full name as described in the data server directory,
URL ID is the 4-char id of the station as specified in the on-line data file and
DSO ID is the 4-char id of the station when downloaded. For example, entry
of the tbl file
002 AgNikolaos agni agnl
will search (for e.g. doy 36 of 2014) for the file ftp://tree-co-data-server/Rinex_URANUS/AgNikolaos/2014/Feb/05/agni0360.14d.Z
and download it to agnl0360.14d.Z

EXIT STATUS

The program will return an integer, denoting the exit status. There are three distinct cases,

•

Exit Status = -2; user has not provided any stations to download

•

Exit Status = -1; this denotes that an error has occured

•

Exit Status >=0 ; this denotes that the program had a seccesseful run and downloaded this many station files. Note that some of the stations may have not actualy been downloaded by this run, but the wanted file(s) was found.

STATION PRIORITY

In case both -s and -f switches are used, the order of the stations is first the -s stations and then the -f ones (e.g > wgeturanus -s sta1 sta2 -f filename ... and filename has stations sta3 sta4, then the list of stations to download will be: sta1 sta2 sta3 sta4). The order of stations is crucial when the -m switch is specified.

DEPENDANCIES

The python library bpepy.gpstime must be installed and set in the python path variable.
