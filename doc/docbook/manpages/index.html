Synopsis

block_amb_read [-f --summary-file=file] [-m --method= method] [-h --help ] [-v --version ]

DESCRIPTION

block_amb_read will read an ambiguity summary file and report a block of ambiguity information in an xml table. I.e. it will create xml table row entries (one for each baseline, plus one for each method --kind of method summary). The format of the summary file is very stringent and applies to output from Bernese v5.2 RNX2SNX output. The xml output is directed to stdout.

OPTIONS

-f --summary-file= file

Takes file i.e. the ambiguity resolution summary file. The ambiguity summary file has a very stringent format and this program is hardcoded to work exactly with this. You can see an example of such a file at: bpe2@vincenty.ntua.gr~/templates/dd/AMBYYDDD0.SUM.

-m --method= resolution_method

Specify an ambiguity resolution strategy to extract. The parameter resolution_method can take one of the following values:

  • 'cbwl' for Code-Based WideLane,
  • 'cbnl' for Code-Based NarrowLane,
  • 'pbwl' for Phase-Based WideLane,
  • 'pbnl' for Phase-Based NarrowLane,
  • 'qif' for QIF,
  • 'dir' for Direct L1/L2

-h --help

Print help message and exit.

-v --version

Print version and exit.

USAGE

block_amb_read -f some_file -m qif

EXIT CODES

  • 0 success, or
  • 1 error

NOTES

  • The output of a program run, cannot be interpreted as a standalone xml document. It is normally used a part of a lager file (e.g. daily summary report of GNSS processing).
  • If the specified method is not reported in the input summary file, then the program will output an empty row (i.e. all entries/columns will be empty).
